04 — UX 设计师 vs AX 设计师：核心能力差异
基于 Google PAIR、Microsoft 等大厂 JD 及前田约翰《Design in Tech 2026》整理，AX 设计师需要掌握的新能力维度：
✓ 国外大厂 AX 设计师已具备
🤖
Agent 行为设计：设计 AI Agent 的决策逻辑、任务分解、错误恢复机制，而非仅设计界面
💬
Prompt 工程能力：能直接编写和优化 System Prompt，理解 LLM 输出的不确定性
🎯
意图对齐设计：从"设计路径"转向"设计意图理解"，让 Agent 准确捕捉用户真实需求
⚖️
AI 伦理与信任设计：设计 AI 透明度、可解释性、用户控制权
🔄
多模态交互设计：覆盖语音、视觉、文本、Agent 自主操作等多种交互模式的统一体验
📊
不确定性体验设计：设计 AI 输出不稳定时的降级方案、置信度展示、用户预期管理
✗ 美团设计师当前缺失
❌
无 Agent 设计方法论：设计师仍以传统"用户操作→界面响应"思维设计 AI 产品，缺乏 Agent 自主性设计框架
❌
无 Prompt 设计能力：设计师与 AI 能力之间存在"翻译层"，无法直接参与 AI 体验的核心决策
❌
缺乏意图设计思维：产品仍以"功能入口"为核心，而非以"用户意图"为核心重构交互逻辑
❌
AI 伦理意识薄弱：无专项 AI 伦理设计规范，用户对 AI 决策的信任感和可控感设计缺失
❌
多模态经验不足：设计师主要聚焦视觉界面，缺乏语音、Agent 自主操作等新交互模式的设计经验
❌
无不确定性设计规范：AI 产品出错或输出不稳定时，缺乏统一的体验降级和用户预期管理方案
